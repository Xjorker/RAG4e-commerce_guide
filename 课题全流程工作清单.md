# 基于 RAG 的多模态电商智能导购 Agent — 课题全流程工作清单

> 来源：字节跳动 AI 全栈挑战赛 | 课题导师：张贺（字节商业化团队）
> 开发周期：2026.05.20 - 06.10（3周）| 答辩：06.11 - 06.20

---

## 一、硬性要求（必须满足）

| 项目 | 要求 |
|------|------|
| 客户端 | ⚠️ **必须原生开发**（Android 或 iOS 任选其一），不接受纯 Web/H5 方案 |
| 后端 | 不限语言框架（Python/Go/NodeJS 均可） |
| 端到端闭环 | 必须跑通：客户端对话 → 后端 RAG 检索 → 模型生成 → 流式返回 → 商品卡片展示 |
| 部署 | 支持私有化部署即可 |

---

## 二、核心工作模块（按开发顺序）

### 🔷 模块1：数据工程与特征治理

- [ ] 1.1 获取官方提供的 ~100 条电商基础数据集
- [ ] 1.2 分析数据集覆盖的品类和属性，找出缺失字段（如材质、成分等）
- [ ] 1.3 扩展数据集——补充缺失属性（如衣服材质、护肤品成分等）
- [ ] 1.4 设计数据结构，覆盖不同场景/类目
- [ ] 1.5 设计向量化方案（选择 Embedding 模型）
- [ ] 1.6 选择向量数据库（本地/云服务均可，如 Milvus/Chroma/FAISS）
- [ ] 1.7 构建向量索引，完成数据入库

### 🔷 模块2：RAG 召回链路设计

- [ ] 2.1 设计意图识别模块（区分不同用户需求走不同链路）
- [ ] 2.2 实现基础检索：关键词/向量相似度检索
- [ ] 2.3 实现混合检索策略（向量 + 关键词）
- [ ] 2.4 设计 Rerank 重排序策略，提升检索准确率
- [ ] 2.5 防幻觉处理：确保生成内容基于检索结果，不编造商品信息
- [ ] 2.6 针对不同场景设计不同的召回链路（推荐/对比/排除等）

### 🔷 模块3：大模型集成

- [ ] 3.1 配置豆包 Seed 模型 API（官方提供，注意不要外泄）
- [ ] 3.2 设计 Prompt 模板（推荐、对比、排除等不同场景）
- [ ] 3.3 实现流式输出（SSE/WebSocket），降低对话延迟感
- [ ] 3.4 上下文管理——多轮对话记忆
- [ ] 3.5 可选：接入其他大模型作为备选

### 🔷 模块4：后端服务开发

- [ ] 4.1 搭建后端框架（FastAPI/Flask/Express/Gin 等）
- [ ] 4.2 实现流式 API 接口（核心对话接口）
- [ ] 4.3 实现向量检索接口
- [ ] 4.4 实现大模型调用接口
- [ ] 4.5 实现会话管理（Session/Context）
- [ ] 4.6 错误处理与异常兜底
- [ ] 4.7 API Key 安全管理（放入 .gitignore，不外泄）

### 🔷 模块5：原生客户端开发（Android 或 iOS）

- [ ] 5.1 搭建原生项目（Android Kotlin/Java 或 iOS Swift）
- [ ] 5.2 实现类豆包的聊天界面
- [ ] 5.3 实现逐字流式渲染（打字机效果）
- [ ] 5.4 实现商品卡片嵌入对话流（无缝展示在聊天气泡中）
- [ ] 5.5 商品卡片支持点击跳转到商品落地页
- [ ] 5.6 对话输入组件（文本输入框）
- [ ] 5.7 与后端流式 API 对接（SSE/WebSocket 客户端）

## 三、场景功能实现（从基础到高级）

### 🟢 基础场景（必须实现）

- [ ] S1 单轮模糊推荐：如"推荐适合油皮的洗面奶" → 意图识别 + 属性匹配
- [ ] S2 基础商品检索与推荐展示

### 🟡 进阶场景（强烈建议实现）

- [ ] S3 条件筛选/多轮追问：如"推荐跑鞋" → "轻量一点" → "500 以内"
- [ ] S4 多轮上下文记忆与对话管理
- [ ] S5 商品对比决策：如"A 和 B 两个面霜哪个更保湿" → 结构化对比

### 🔴 高级场景（加分项，深度优先）

- [ ] S6 反选/排除：如"防晒霜，不要含酒精的，不要日系品牌"
- [ ] S7 场景化组合推荐：如"下周去三亚度假" → 理解场景 → 组合推荐
- [ ] S8 购物车操作：通过对话实现购物车增删改查
- [ ] S9 下单流程闭环
- [ ] S10 拍照找货（多模态）：上传图片 → 图像理解 → 相似商品检索
- [ ] S11 语音输入（多模态）：语音识别 → 文本 → 检索推荐

## 八、开发策略

> 导师原话：**"先跑通最小闭环，再逐步增强"**

1. **第一步**：跑通 原生客户端 → 后端 → LLM → 流式返回 的最小链路
2. **第二步**：接入 RAG 检索，实现基础推荐场景
3. **第三步**：完善商品卡片展示 + 落地页跳转
4. **第四步**：实现多轮对话 + 进阶场景
5. **第五步**：选择一个加分方向做深

---

## 九、注意事项

1. **API Key 安全**：豆包模型 API Key 仅限课题内使用，不要外泄，上传 GitHub 时放入 .gitignore
2. **AI 辅助开发**：鼓励使用 AI 开发，但必须主导架构设计，答辩时会针对性提问实现原理
3. **理解原理**：使用现成 AI 框架可以，但要理解内部原理
4. **数据集扩展**：官方数据集只有 ~100 条且属性有限，需要根据场景自行扩展
5. **录制保底视频**：答辩时可能因各种原因跑不起来，提前录制演示视频
