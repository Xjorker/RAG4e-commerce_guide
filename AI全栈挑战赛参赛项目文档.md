## 1. 项目简介
本项目实现了一套完整的端到端AI智能导购系统，面向电商购物场景，采用RAG检索增强生成架构，结合多维度语义过滤引擎，支持自然语言对话选品、图片搜索找同款、智能推荐商品对比、SSE实时流式回复、一键加购等完整购物流程。

## 2. 系统架构 & 技术栈

### 2.1 架构图
```
Android移动端 (Jetpack Compose)
       ↓ SSE / RESTful API
FastAPI 后端服务 (Python 3.10+)
       ├─ 认证模块 (JWT Token)
       ├─ SSE 流式对话入口
       ├─ V4.3 全维度意图理解引擎
       │  ├─ 子品类识别
       │  ├─ 品牌过滤
       │  ├─ 绝对/相对价格
       │  ├─ 颜色匹配
       │  └─ 功能属性语义召回
       ├─ 全量历史结果上下文缓存
       ├─ BM25 关键词检索引擎
       ├─ LLM 大模型服务 (通义千问/DeepSeek)
       ├─ 向量数据库 (Chroma) 图片特征检索
       └─ SQLite 持久化存储
              └─ 100款真实商品 + 585个多规格SKU
```

### 2.2 技术栈清单
| 分层 | 技术选型 |
|------|----------|
| 后端框架 | FastAPI + Python 3.10+ |
| 数据库 | SQLite 3 (内置，无需额外部署) |
| 向量检索 | Chroma + CLIP 图像特征提取 |
| 文本检索 | BM25 (Okapi) 全量商品知识库 |
| 移动端 | Android Jetpack Compose (Kotlin) |
| 网络协议 | SSE 服务器推送事件 (流式回复) |

## 3. 核心亮点/创新点

### 创新点1：全维度历史继承式意图理解引擎V4.3
不是每次查询都重新检索全库，而是基于上一轮的推荐结果，在上下文里做渐进式过滤：
- "15000以内笔记本" → 返回5个笔记本
- "不要华为的" → 仅从5个里面剔除华为，保留其他4款，不会把全库100个商品都返回出来
- "更便宜一点的" → 自动继承上下文，直接返回价格向下浮动30%的结果，用户无需重复描述任何约束

### 创新点2：SSE流驱动自动加购
大模型在流式回复过程中识别用户"加购物车"意图，无需跳转页面，直接在对话流中完成购物车添加，完全自然交互。

### 创新点3：全量真实多SKU电商数据集
100款4大品类商品 + 585个真实多规格SKU，覆盖笔记本/手机/美妆/服饰/食品，每款商品都有完整的详情RAG知识库，不是Demo级别的模拟数据。

## 5. 快速部署体验指南

### 步骤1：启动后端
```bash
# 安装依赖
pip install -r requirements.txt
# 启动服务 (默认端口 8000)
python start_fastapi.py
```

### 步骤2：启动Android App
1. 在 Android Studio 打开 `RAG-Shopping-Guide-Android` 项目
2. 配置网络安全配置允许本地 10.0.2.2:8000 访问后端
3. Run 安装到模拟器即可直接使用

### 3条快速体验路径
1. 自然语言："15000以内的笔记本，不要华为的" → 系统正确返回4款非华为笔记本卡片
2. 功能属性："找一双有缓震的跑步鞋" → 精准召回带气垫/碳板的跑鞋
3. 图片搜款：点击附件按钮，选择任意数码产品实拍图，系统自动用CLIP向量匹配找同款
4. 加购物车：对AI说"把第一款 MacBook Pro 14英寸的给我加购" → 自动识别加入购物车

## 6. 关键问题解决方案
| 问题点 | 解决方式 |
|--------|----------|
| 商品卡片不显示 | 移除Android端冗余的本地二次分类过滤，完全信任后端返回结果 |
| "不要华为"返回全库 | 引入历史结果缓存机制，纯继承查询仅在上一轮返回集合中操作 |
| 多SKU规格丢失 | 重写全量数据导入脚本，585真实SKU完整入库 |
| 对话响应慢 | 改用SSE流式输出，大模型每生成几个Token就推送到前端 |
